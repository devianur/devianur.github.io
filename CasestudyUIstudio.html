DESIGN PROCESS OF LEARNING ONLINE FLATFORM VIA WEBSITE

- UI/UX STUDIO -

OVERVIEW

UI/UX Studio is UI/UX learning via website. This platform has many feature for develop skill in UI/UX Design. Available for free course and fee course. Surely, there are many different feature for both. One of them, there is feature Applying job and job invitation for paid account. This features available based on problem statement that i found in user need.

Please check below for more details :

01
DISCOVER

- Assumption Hyphotesis-

Before let in further steps, I make assumption/ hyphotesis , what the possibility of user needs. I used frame of “ the product vision board” to detailing every assumption that i made.

PRODUCT VISION BOARD
VISION
TARGET GROUP
  • Individual/mahasiswa yang memiliki keinginan menjadi UX Designer.
  • Calon karyawan junior UX Designer
  • Perusahaan yang membutuhkan karyawan sebagai UX Designer

NEEDS
  • Menyediakan access pembelajaran untuk menjadi UX Designer
  • Bergabung dalam komunitas UX Designer
  • Mendapatkan informasi mengenai lowongan pekerjaan
  • Mendapatkan kesempatan peluang tawaran dari perusahaan yang membutuhkan UX Designer
PRODUCT
  • Sebuah platform yang berisi materi pembelajaran bagi calon UX Designer. (Free for limited access)
  • Terdapat akses dalam melakukan pelatihan dengan adanya penugasan (for purchased)
  • Terdapat forum diskusi yang berisi peserta dan tutor / senior UX designer untuk sharing dan bertanya. (for purchased)
  • Memberikan fitur apply job dan informasi lowongan pekerjaan di bidang UX Designer.(for purchased)
  • Adanya fitur invitation dari perusahaan yang berniat mengundang peserta tertentu untuk bergabung menjadi karyawannya.(for purchased)
BUSINESS GOALS
  • Mendapatkan commision revenue dari peserta yang mengikuti pelatihan
  • Branding
  • Memiliki partner dari berbagai perusahaan
  • Mendapatkan commission revenue dari perusahaan yang ingin menempatkan iklan lowongan pekerjaannya di pencarian teratas .

02
DEFINE

- User Research -

After i made hyhpotesis, The second process is Define. There are 3 Part in define, and one of them is User Research. User research process needs to know what user needs, what the problems of user feels. In this process, interview to users is the step that i choose to confirm my hypotesis , get empathy and all i need to find the others problem that i didn’t get before.

1. User Interview

I get two users for interview. There are man and woman that interested with UI/UX Designer. From this interview, i made some of questions before interview to users . The important points of question are what user needs, what goals of the user expects and what the points made user frustations when they studied UI/UX design.

2. Affinity Mapping

After collecting the result from the user interviews, I conducted affinity mapping to synthesise the problems and get the insight of madlibs. We grouped these problems based on the same criteria.

3. Insight Madlibs

The criteria was grouped are :

  1. User Needs
  2. Price of course
  3. Time for study
  4. Matery Needs
  5. Goals expectation
  6. The reason why must learn UI/UX Design
  7. Paint of points
USER NEEDS INSIGHT
  • Mahasiswa dan juga mahasiswa fresh graduate yang tertarik dengan UI/UX design dan berniat untuk bekerja menjadi seorang UI/UX Designer
  • Pemahaman UI / UX Designer yang didapat dari kursus dengan mentor yang baik
  • Kursus yang flexible waktu dan tempat
  • Kursus dengan harga murah
  • Membutuhkan materi yang mudah dipahami, runtut, termasuk video pembelajaran dan komunitas untuk bertukar informasi
  • Kesempatan untuk berkarir sebagai UI/UX Designer
  • User ingin mengembangkan pengetahuan dan skill di dunia UI/UX Designer , walaupun tidak memiliki kemampuan mendesign dengan baik.
  • Membutuhkan info loker ,berharap memiliki peluang atau kesempatan untuk berkarir sebagai UI/UX Designer

From all the result, the insight of madlibs is :
"A student of college or fresh graduate who does not have good design skills, has an interest about UI / UX Designers, so they need learning facilities with qualified teacher but the prices is relative affordable. The material of learning is easy, complete , coherent and have interesting learning videos, so they have the opportunity to have a career as a UI / UX Designer.

4. User Persona

In order to empathize with the different types of users, I made so-called user personas, which are based purely on experience and assumptions. These serve as a starting point to reflect on the needs and challenges of the user and possible solutions that can be considered later in building the information architecture.

5. User Journey

In the user journey, we can know phase journey of user to get their goals. and through the problems.User Journey or so-called journey map always visualize the user's journey / process and things related to the 'journey', such as showing the behavior of the user, his emotions, channel or touchpoint interactions, and showing the linear timeline he experienced. This information used by designer to get the design process for their solutions. These are user journey that i got.

03
IDEATE PHASE

In this process, the solution was made. The problems had been got before can be refference to found the solution. How the platform was made, for whom application was made and what the main point that must application had. Not only about that, but we can know how the user flow will be made.
The solution is, making web application for UI/UX learning that have :

  1. Material completely and can be access in flexible time
  2. Tutors with great experience
  3. Video tutorial
  4. Application available for free mode (not purchased, but limited)
  5. Application available for fee mode
  6. There is a group ui/ux for discuss (Purchased)
  7. There is menu apply job for member (purchased)
  8. There is chance to get invitation a job for member from a company (purchased)

04
SKETCH

After list the idea, i sketch the mockup in a paper to visualize it in in wireframe later. I sketch the button, frame, and menu. There are home, Programs, Register and Login menu. Register menu has a text field to payment code as a requirements to get account.

05
WIREFRAME

The sketch was made before, i use to made wireframe with figma. I completely details and add the wireframe or menu that i was not made in paper .

06
PROTOTYPE

Prototype is simulation of all idea that was developed before code with development. There are many interaction in every pages, so user can simulated before. If there is something wrong in the prototype, i can redesign the application. In this prototype i choose the green and white for background to describe the pschology of growth,harmony, dependability, and it match for industri in education.

CONCLUSION

Due to any process which is not short, design of UI/UX learning application had been done. With through the user reasearch, application solution found some point that not imagined by designer before.

As a learning :
User interview is a must, but don’t forget about another stakeholder, such as employee or staff of a company, because they have to join in this web to give or input information about recruitment in their company, not only about that, they must able to access the web to see all the members and choose the member who have criteria as follow their needs to be employee in their company.
And, if there was more time, I would make more menu and user reasearch in competitor.
